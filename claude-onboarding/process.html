이 흐름이 강의 사례인 이유

STEP 1

1차 LLM으로 콘텐츠 생성

ChatGPT 5.5에 강의 의도와 청중을 설명하고, 강의안·데이터셋·Skill 예시를 한 번에 받는다. 단일 가상 시나리오로 묶어 일관성 확보.

STEP 2

2차 LLM으로 검토·거버넌스·시각화

Claude Opus 4.7에 폴더 전체를 넘기고 "검토 의견 → 향후 계획 → 구현"을 한 호흡에 요청. 거버넌스 4종으로 다음 세션이 컨텍스트 재구성을 안 하도록 잠금.

STEP 3

3차 이후 LLM이 이어받게

rule.md + AGENT.md + 누적 log.md가 다음 에이전트(또는 미래의 본인)의 진입점이 된다. 같은 폴더가 1회성 산출물에서 재사용 가능한 "강의 제작 Skill"로 바뀐다.

토큰 추정의 한계

  • 모든 수치는 추정이다. 실제 API 청구는 모델별 토크나이저 결과로 결정된다.
  • Claude의 캐시 히트(반복 컨텍스트)는 청구 토큰을 줄여주지만 여기 표에는 반영하지 않았다.
  • ChatGPT 측 사용자 화면에는 토큰이 직접 노출되지 않으므로, 세션 1 추정은 입출력 본문 길이 기준 환산이다.
  • Claude Code 시스템 프롬프트는 약 18K~22K 토큰 규모로, 한 세션 내 모든 요청에 매번 동반된다(캐시 적용 시 청구 토큰은 더 작음).
  • "실작업 시간"은 사람이 응답을 읽고 다음 프롬프트를 쓰는 시간을 포함한다. 모델 응답 시간만 따지면 더 짧다.
